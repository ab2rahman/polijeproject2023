Mobile Application Project POLIJE (Kotlin)

Create Seven Pages + One Bonus Page (Optional): Splash Screen Login Page Register Home Profile Favorite Detail User Motion Layout (Bonus)

You can access Prototype from Here : FIGMA PROTOTYPE LINK

  • Splash Screen *must have

    1. Logo can be anything
    2. Give a three-second (3000ms) delay to automatically go to the next page
    3. Check whether you are logged in or not using DataStore Preference
      If not then you will be directed to the Login page or to the Onboarding Motion Layout (bonus) page
      If so, you will be directed to the Home page
    4. Tutorial How to Create Splash Sceen

  • Login Page *must have

    1. Create a Login page
    2. User can log in
    3. User can click the register button to move to the register page
    4. Check whether the inputted username and password are saved on Firebase Auth
    5. If successful in logging in, he will be directed to the dashboard page and state "logged in" will be saved in DataStore Preference

  • Register Page *must have

    1. Create a Register page to register an account (can be within the same page as the Login page or create a new Page)
    2. At least there is an app username, password, GitHub username, nim, and email
    3. Data stored in the Firebase Auth

  • Home Page *must have

    1. Create a Home Page (can be Activity or Fragment)
    2. Display a list of users from GitHub API, you can read this Documentation, and using Retrofit
    3. + Bonus Points if you can use Paging for the recyclerView, you can learn from this Tutorial
    4. Add favorite function to save user from the recyclerView to Room DB
    5. Can be searched using searchview

  • Detail User Page *must have

    1. Create a Detail User Page (can be Activity or Fragment)
    2. Pass Argument or Intent from Home, using git username to get user data from GitHub API
    3. Show user data from API Response

  • Profile Page *must have

    1. Create a User Page (can be Activity or Fragment)
    2. Show the user login which show us detail user GitHub (from logged-in git username) from GitHub API
    3. Also, include a logout function to move to the Login page so that other users can log in / register

  • Favorite Page *must have

    1. Create a Favorite Page (can be Activity or Fragment)
    2. Show the list of git user that saved as favorite from home page that saved on Room DB
    3. Add function to unfav the user from list

  • OnBoarding Motion Layout Page (bonus)

    1. Create an OnBoarding Page using Motion Layout
    2. Create minimum of three board, the content can be anything
    3. Tutorial OnBoarding Project
